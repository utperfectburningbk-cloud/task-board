# タスク管理ボード

設計チーム向け、iPad で使うタスク管理ツール。緊急度 × 重要度の4象限で「いま何をやるべきか」を
一目で把握することを最優先に設計しています。

- 元デザイン: Claude Design プロジェクト「iPad横向きデザイン仕様確認」の `タスク管理ボード.dc.html`
- 要件書: 同プロジェクトの `uploads/task_manager_design_brief.md`

各メンバーが**自分専用のボード**を持ちます。データは端末のブラウザ内に保存され、
他メンバーのボードは見えません（相互共有は要件上スコープ外）。

## メンバー向け：導入手順（iPad）

1. Safari で上記の URL を開く
2. 共有ボタン <kbd>􀈂</kbd> →「**ホーム画面に追加**」
3. ホーム画面のアイコンから起動する
4. 最初は空のボードです。「＋タスク」から案件とタスクを登録してください
   （案件は最初のタスクを追加するときに、その場で作れます）

> **手順2は必須です。** Safari のタブのまま使い続けると、しばらく開かなかったときに
> iOS がサイトデータを自動削除し、タスクが消えることがあります。
> ホーム画面に追加したアプリはこの削除の対象外になります。あわせてオフラインでも起動できます。

### 使い方

| 操作 | やり方 |
|---|---|
| タスクを追加 | 右上「＋タスク」→ ①案件 → ②タスク名 → ③緊急度・重要度＋期限日 |
| タスクを編集 | カードをタップ |
| タスクを完了 | カード右下の丸をタップ。取り消し線が付き、数秒後にアーカイブへ移動（その間は「取り消し」可） |
| タスクを削除 | カードを**左にスワイプ**（直後に「取り消し」可） |
| 案件で絞り込む | 上部「▽ 絞り込み」。複数選択可、未選択なら全案件 |
| 手書きする | 上部「✎ 手書き」。ボード全体に重なる透明レイヤーに書けます。オフにすると非表示 |
| 完了タスクを見る | 上部「🗄」。「戻す」でボードに復帰 |
| 表示名・バックアップ | 上部「⚙」 |

### バックアップ（大事）

タスクは**この端末のブラウザ内だけ**にあります。次のときは必ず「⚙ → 書き出す（JSON）」を
実行して、ファイルアプリや iCloud Drive に保存してください。

- iPad を買い替える・初期化するとき
- Safari の「履歴とWebサイトデータを消去」をする前

復元は「⚙ → ファイルから復元」で、書き出した JSON を選ぶだけです。
（復元すると、いまボードにあるタスクは置き換えられます）

## 管理者向け：GitHub Pages への展開

**設定済みです。** 以下は構成の記録と、作り直すときの手順です。

| 項目 | 値 |
|---|---|
| リポジトリ | https://github.com/utperfectburningbk-cloud/task-board （Public） |
| 公開URL | https://utperfectburningbk-cloud.github.io/task-board/ |
| Pages ソース | ブランチ `main` / ルート `/` |

> **Public リポジトリである点について。** 無料プランでは Private リポジトリから Pages を
> 公開できないため Public にしています。ソースは誰でも読めますが、パスワードや顧客データは
> 一切含まれません（タスクは各端末の localStorage のみ、サーバーには何も送信しない）。
> 初期データも空なので、案件名が外部に出ることもありません。
> **案件名や個人情報をソースに直接書かないでください。**

### 更新するとき

**`index.html` の `APP_VERSION` と `sw.js` の `CACHE` を同じ新しい値に上げてから** push します。

```bash
git add -A
git commit -m "○○を修正"
git push
```

各メンバーの端末では、次にアプリを開いたときに画面下へ
「新しいバージョンがあります [更新]」が出ます。タップすると切り替わります。
バージョンを上げ忘れると Service Worker が古いキャッシュを配り続け、更新が届きません。

#### 更新でタスクが消えないのはなぜか

アプリの本体（HTML/JS/アイコン）と、各メンバーのタスクは**別々の保存領域**にあります。

| | 保存先 | 更新時の扱い |
|---|---|---|
| アプリ本体 | Cache Storage（`task-board-v<バージョン>`） | 古い版を削除し、新しい版に入れ替える |
| タスク・案件・表示名・手書き | localStorage（`lrt-task-board-v1`） | **一切触らない** |

Service Worker の `activate` が削除するのは Cache Storage だけで、localStorage には手を出しません。
v1.1.0 → v1.2.0 の更新を実際の公開URLで通し、更新前に登録したタスク・案件・表示名が
そのまま残ることを確認済みです。

**ただし `STORE_KEY`（`lrt-task-board-v1`）を変更すると、全員のデータが参照されなくなります。**
保存する項目を増やすときは、キー名は変えずに `snapshot()` / `load()` にフィールドを足してください。
`load()` は知らないフィールドを無視し、欠けているフィールドは既定値で補うため、
新旧バージョンが混在しても壊れません。

#### データを読めない・保存できないとき

- 保存データが壊れて読めない場合、中身を `lrt-task-board-v1.recover.<timestamp>` へ退避してから
  空のボードで起動し、画面上部に警告を出します。**元データを上書きすることはありません**
- 容量超過やプライベートブラウズで保存できない場合も、警告帯で明示して書き出しを促します
  （黙って失敗させると「入力したのに保存されていない」ことに気づけないため）

### ローカルで動かす

```bash
python -m http.server 8941 --directory E:/LRT/task-board
# → http://localhost:8941
```

Service Worker は `http(s)://` でのみ動きます（`file://` で直接開いた場合は自動的に無効化され、
オフライン動作と更新通知だけが使えません。アプリ本体は動きます）。

## 技術メモ

### ファイル構成

```
task-board/
├── index.html      … アプリ本体（HTML/CSS/JS すべて。ビルド不要・依存なし）
├── sw.js           … Service Worker（オフライン動作・更新配布）
├── manifest.json   … ホーム画面に追加したときの名前・アイコン・全画面表示
└── icons/          … apple-touch-icon.png / icon-192.png / icon-512.png
```

### データ

- 保存先: `localStorage` キー `lrt-task-board-v1`（端末・ブラウザごとに独立）
- 中身: `owner`（表示名）/ `projects` / `tasks` / `ink`（手書きレイヤーの dataURL）/ `backedUpAt`
- タスク: `{ id, pj(案件id), t(タスク名), u(緊急 0|1), i(重要 0|1), due(YYYY-MM-DD), done, arch }`
- 初回起動時は**空のボード**（サンプルデータなし）。案件は最初のタスクを追加するときに作る

### 主な調整ポイント（`index.html` 冒頭の定数）

| 定数 | 既定 | 内容 |
|---|---|---|
| `CARD_CAP` | 4 | 各象限に表示するカード数の上限。超過分は「＋n件を表示」で折りたたむ |
| `ARCHIVE_DELAY` | 4200 | 完了してからアーカイブへ移動するまでの ms。「取り消し」を出す猶予 |
| `TOAST_DELAY` | 4600 | トーストを表示しておく ms |

### 期限の色分けルール

| 状態 | 判定 | 表現 |
|---|---|---|
| 超過 | 期限日を過ぎた | 赤系の背景・枠線＋「n日超過」バッジ |
| 本日 | 期限日が今日 | 橙系の背景・枠線＋「本日」バッジ |
| 間近 | 3日以内 | 黄系の背景・枠線＋「あとn日」バッジ |
| 通常 | それ以降 | 無彩色 |

案件色はカード左端のカラーバー、期限強調は背景・枠線と、役割を分けています。

### デザイン原本からの変更点

- iPad のベゼル枠と向き切替ボタンを削除（モックアップ用の装飾のため）。実機の画面サイズに追従し、
  幅 900px 以下でボタンのラベルを畳んでアイコンのみにします
- 日付を固定値から実日付に変更
- localStorage 永続化、バックアップの書き出し／復元、表示名、Service Worker を追加
